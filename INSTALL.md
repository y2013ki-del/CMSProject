  1. 사전 준비
================================================================

  [ Node.js ]
  - https://nodejs.org 에서 LTS 버전 설치
  - 확인: CMD에서 node -v

  [ FFmpeg ] (CCTV 스트리밍 사용 시)
  - https://ffmpeg.org/download.html 에서 Windows 빌드 설치
  - C:\ffmpeg\bin\ffmpeg.exe 경로에 배치
  - 확인: CMD에서 ffmpeg -version

  [ 네트워크 요구사항 ]
  - 서버 PC와 QMC 디스플레이가 동일 네트워크에 있어야 함
  - 서버 PC는 고정 IP 설정 권장 (DHCP는 IP 변경 위험)
  - 방화벽에서 8080 포트(TCP) 인바운드 허용 필요


================================================================
  2. 서버 설치 및 설정
================================================================

  [ 설치 ]

  1) 프로젝트 폴더를 서버 PC에 복사
     예) D:\cms-server\

  2) CMD(관리자)에서 프로젝트 폴더로 이동
     cd D:\cms-server

  3) 패키지 설치
     npm install

     ※ 사내망 보안으로 인터넷 차단된 경우
        - 외부 PC에서 npm install 후 node_modules 폴더 복사

  [ IP 설정 — 필수 ]

  server.js 10번째 줄:
  ─────────────────────────────────────
  const PROXY_IP = '192.168.1.100';   ← 이 서버 PC의 실제 IP로 변경
  ─────────────────────────────────────
  예) const PROXY_IP = '10.10.1.50';

  실제 IP 확인: CMD에서 ipconfig → IPv4 주소 확인

  [ 서버 실행 테스트 ]
  node server.js

  정상 출력:
    CMS 서버 실행 중: http://localhost:8080
    플레이어 앱:       http://192.168.1.100:8080/player?channel=<채널ID>
    RTSP 프록시:      http://localhost:8080/stream/<카메라명>

  브라우저에서 http://localhost:8080 접속
  → 로그인 화면 확인 후 Ctrl+C로 종료

  [ 로그인 정보 (초기값) ]
  - 아이디:   admin
  - 비밀번호: !!@@password


================================================================
  3. 자동 시작 설정 (Windows 작업 스케줄러)
================================================================

  PC 부팅 시 자동으로 서버가 실행되도록 등록.
  관리자 CMD에서 실행.

  [ 등록 ]
  schtasks /create /tn "CMS-Server" /tr "node D:\cms-server\server.js" /sc onstart /ru SYSTEM /f

  ※ 경로는 실제 설치 경로로 변경
  ※ 성공 시: 성공: 예약된 작업 "CMS-Server"이(가) 만들어졌습니다.

  [ 수동 켜기 / 끄기 ]
  켜기:   schtasks /run /tn "CMS-Server"
  끄기:   schtasks /end /tn "CMS-Server"
  상태:   schtasks /query /tn "CMS-Server"
  확인:   tasklist | findstr node
  삭제:   schtasks /delete /tn "CMS-Server" /f

  [ 간편 BAT 파일 생성 (선택사항) ]

  start_cms.bat:
    schtasks /run /tn "CMS-Server"
    echo CMS 서버 시작됨
    pause

  stop_cms.bat:
    schtasks /end /tn "CMS-Server"
    echo CMS 서버 중지됨
    pause

  4. QMC 사이니지 — 플레이어 설치 및 실행
================================================================

  ┌─────────────────────────────────────────────────────────┐
  │  QMC 플레이어 URL 형식                                   │
  │  http://[서버IP]:8080/player?channel=[채널ID]            │
  │                                                         │
  │  채널 ID는 관리자 UI → 스케줄 탭에서 확인               │
  └─────────────────────────────────────────────────────────┘

  ─────────────────────────────────────────────────────────
  방법 1: URL Launcher (권장 — 가장 단순, 업데이트 즉시 반영)
  ─────────────────────────────────────────────────────────

  1) QMC 리모컨으로 설정 진입
     홈 버튼 → 설정 → 일반 → 시스템 관리자 메뉴
     ※ 모델에 따라 경로가 다를 수 있음

  2) URL Launcher 설정
     시스템 관리자 메뉴 → URL Launcher 설정 → URL Launcher ON

  3) 플레이어 URL 입력
     URL 입력 → http://192.168.1.100:8080/player?channel=ch_xxxxxxxx

  4) 저장 후 재부팅
     → 부팅 시 자동으로 플레이어 앱 실행

  ※ 앱 업데이트: server.js/player/index.html 수정 후 QMC 재부팅
     또는 리모컨으로 새로고침 (F5 또는 홈→앱 재실행)

  ─────────────────────────────────────────────────────────
  방법 2: MagicInfo 없이 앱 관리 메뉴로 설치
  ─────────────────────────────────────────────────────────

  일부 QMC 모델에서 사용 가능:

  1) 홈 버튼 → 앱 → 앱 관리
  2) 오른쪽 상단 "설치 (URL)" 또는 "Install Custom App"
  3) URL 입력: http://192.168.1.100:8080/player?channel=ch_xxxxxxxx
  4) 설치 완료 후 앱 목록에서 실행

  ─────────────────────────────────────────────────────────
  방법 3: USB 설치 (SSSP 6 이하 모델)
  ─────────────────────────────────────────────────────────

  1) USB 드라이브 루트에 SSSP 폴더 생성
  2) player/index.html을 SSSP/index.html로 복사
  3) index.html 내 serverBase 값을 실제 서버 IP로 수정:
     var serverBase = 'http://192.168.1.100:8080';
     (기존: location.protocol + '//' + location.host)
  4) QMC에 USB 삽입 → 자동 감지 → 설치 진행

  ─────────────────────────────────────────────────────────
  채널 ID 확인 방법
  ─────────────────────────────────────────────────────────

  1) 관리자 UI → http://서버IP:8080 접속
  2) 스케줄 탭 → 채널 생성 또는 기존 채널 선택
  3) 브라우저 개발자 도구(F12) → Network 탭 →
     /api/schedules 응답에서 "id" 값 확인
     예: "id": "ch_a1b2c3d4"

  또는: 서버 PC에서 data/schedules.json 파일을 텍스트 편집기로 열어 확인

  ─────────────────────────────────────────────────────────
  여러 QMC에 다른 채널 배정
  ─────────────────────────────────────────────────────────

  TV 01 → http://서버IP:8080/player?channel=ch_aaaa1111  (채널 1)
  TV 02 → http://서버IP:8080/player?channel=ch_bbbb2222  (채널 2)
  TV 03 → http://서버IP:8080/player?channel=ch_aaaa1111  (채널 1과 동일)

  ─────────────────────────────────────────────────────────
  플레이어 디버그 모드 (설치 전 PC에서 테스트용)
  ─────────────────────────────────────────────────────────

  player/index.html 7번째 줄:
  var DEBUG = false;  →  var DEBUG = true;  로 변경

  → 화면 하단에 WebSocket 연결 상태 / 현재 재생 항목 표시


================================================================
  5. 채널 구성 전체 흐름
================================================================

  Step 1. 서버 실행
    → node server.js (또는 자동 시작 등록)

  Step 2. 관리자 UI 접속
    → http://서버IP:8080
    → 로그인 (admin / !!@@password)

  Step 3. 콘텐츠 업로드
    → 콘텐츠 탭 → 파일 드래그앤드롭 또는 클릭하여 업로드
    → 이미지(JPG, PNG), 동영상(MP4, MOV) 지원

  Step 4. 채널 생성 및 스케줄 구성
    → 스케줄 탭 → + 추가 버튼으로 채널 생성
    → 채널 선택 → 미디어 추가 버튼으로 콘텐츠 추가
    → 각 항목의 표시 시간(초) 설정
    → 저장 버튼

  Step 5. QMC에 URL 입력
    → 채널 ID 확인 후 URL Launcher에 입력
    → http://서버IP:8080/player?channel=ch_xxxxxxxx

  Step 6. 디스플레이 등록 (TV 제어 사용 시)
    → 디스플레이 탭 → IP / 이름 / 위치 입력 → 채널 배정
    → 전원 ON/OFF, 입력 전환, 볼륨 제어 가능

  ─────────────────────────────────────────────────────────
  콘텐츠 즉시 변경 방법
  ─────────────────────────────────────────────────────────

  스케줄 탭 → 채널 선택 → 내용 수정 → 저장
  → 저장 즉시 WebSocket으로 모든 플레이어에 자동 반영
  → QMC 재부팅 불필요


================================================================
  6. 서버 관리 명령어
================================================================

  [ 서버 제어 ]
  schtasks /run /tn "CMS-Server"       → 서버 시작
  schtasks /end /tn "CMS-Server"       → 서버 중지
  schtasks /query /tn "CMS-Server"     → 실행 상태 확인
  tasklist | findstr node              → node 프로세스 확인

  [ 로그 확인 ]
  type D:\cms-server\data\cms.log      → 전체 로그 출력
  관리자 UI → 로그 탭                  → 웹에서 확인 (권장)

  [ 데이터 파일 위치 ]
  D:\cms-server\data\cameras.json     → CCTV 카메라 목록
  D:\cms-server\data\displays.json    → QMC 디스플레이 목록
  D:\cms-server\data\schedules.json   → 채널/스케줄 데이터
  D:\cms-server\data\cms.log          → 이벤트 로그

  [ 미디어 파일 위치 ]
  D:\cms-server\media\library\        → 업로드된 콘텐츠

  7. 문제 해결
================================================================

  [ 서버 접속 안 될 때 ]
  - tasklist | findstr node 로 실행 여부 확인
  - 미실행 시: schtasks /run /tn "CMS-Server"
  - Windows 방화벽 → 인바운드 규칙 → TCP 8080 허용 확인
    제어판 → Windows Defender 방화벽 → 고급 설정
    → 인바운드 규칙 → 새 규칙 → 포트 → TCP → 8080

  [ QMC에서 플레이어가 열리지 않을 때 ]
  - QMC와 서버가 같은 네트워크인지 확인
  - QMC에서 서버 IP ping 가능한지 확인
  - URL에 채널 ID가 올바른지 확인 (ch_ 로 시작)
  - server.js의 PROXY_IP가 실제 서버 IP와 일치하는지 확인
  - 서버 재시작 후 재시도

  [ 플레이어에서 콘텐츠가 안 나올 때 ]
  - 관리자 UI → 스케줄 탭에 해당 채널에 콘텐츠가 있는지 확인
  - 콘텐츠 탭에 파일이 업로드되어 있는지 확인
  - 관리자 UI → 스케줄 탭 → 즉시 푸시 버튼 클릭

  [ WebSocket 연결 안 될 때 (DEBUG=true 상태에서 빨간 점) ]
  - 브라우저/QMC에서 ws:// 프로토콜 허용 여부 확인
  - 서버가 실행 중인지 확인
  - 5초 후 자동 재연결 시도함 → 잠시 대기

  [ 동영상이 재생되지 않을 때 ]
  - 파일 형식 확인: QMC는 H.264 MP4 권장
  - H.265(HEVC)는 SSSP 7 이상에서 지원
  - 파일 손상 여부 확인: PC 브라우저에서 http://서버IP:8080/media/파일명 직접 재생 테스트

  [ CCTV 영상이 안 나올 때 ]
  - CMD에서 ffmpeg -version 으로 설치 확인
  - C:\ffmpeg\bin\ffmpeg.exe 경로 파일 존재 여부 확인
  - 카메라 IP/ID/PW 정보 재확인
  - 관리자 UI → 로그 탭 → 스트림 필터로 오류 이벤트 확인

  [ MDC TV 제어가 안 될 때 ]
  - TV IP가 올바른지 확인 (디스플레이 탭에 등록된 IP)
  - TV가 같은 네트워크에 있는지 확인
  - 일부 QMC 모델은 설정에서 MDC 포트(1515) 활성화 필요:
    설정 → 일반 → 네트워크 → 서버 네트워크 설정 확인
  - 로그 탭 → 제어 필터에서 MDC 오류 내용 확인
