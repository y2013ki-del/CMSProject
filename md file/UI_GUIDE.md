# UI_GUIDE — 관리자 UI / 플레이어 UI 기준

> 읽는 시점: 화면 구조, UX 기준, 콘텐츠 디자이너/로그/채널 동작을 확인할 때
> 관련 문서: `README.md`, `md file/PROJECT_STATUS.md`, `md file/API_REFERENCE.md`

## 공통 UI 원칙

1. 운영자가 의미를 바로 이해할 수 있는 이름을 사용
2. 저장/삭제/즉시 반영처럼 결과가 큰 동작은 팝업으로 재확인
3. 다크모드 기준에서도 밝은 잔색이 남지 않도록 유지
4. 콘텐츠는 단일 파일보다 `배치형 편집` 기준으로 다룸
5. 기기 식별은 기본적으로 IP 중심으로 설명

## 관리자 탭 구조

현재 상단 탭은 아래 순서입니다.

1. `기기 모니터링`
2. `기기 제어`
3. `콘텐츠`
4. `스케줄`
5. `채널`
6. `로그`

## 기기 모니터링

- 상단 통계 카드: 전체/온라인/오프라인/미등록
- `접속 감지`: 아직 등록되지 않은 접속 기기 표시
- `기기 모니터링`: 등록된 기기 카드 표시
- 카드에서 확인 가능한 정보
  - 기기명
  - IP
  - 구역
  - 온라인 상태
  - 채널
  - MDC 상태 캐시

## 기기 제어

- 선택 기기 기준 일괄 제어
- 전원, 입력, 볼륨, 재시작, 플레이어 재로드
- 제어 결과는 패널과 toast 둘 다 제공

## 콘텐츠 탭

### 현재 핵심 방향

콘텐츠 탭은 단순 파일 창고가 아니라 `콘텐츠 제작/재편집` 탭입니다.

### 좌측

- 콘텐츠 그룹 목록
- 전체 파일 그룹은 항상 맨 위

### 우측

- 콘텐츠 카드 목록
- 각 카드에서 `수정`, `삭제` 가능
- 일반 파일도 수정 시 이름 변경만 하는 것이 아니라 디자이너 편집으로 진입 가능해야 함

### 콘텐츠 등록 디자이너

상단 툴바:

- 방향: `가로 / 세로`
- 분할: `단독 / 2분할 / 3분할 / 4분할 / 영역 추가`
- 영역 비율: `16:9 / 4:3 / 9:16 / 3:4`
- 소스 추가: `파일 선택 / 웹 콘텐츠 등록 / 날씨 콘텐츠 추가`

좌측 패널:

- 업로드 파일 / 웹 / 날씨 소스 목록
- 드래그앤드롭으로 슬롯 배치

우측 스테이지:

- 슬롯 선택
- 슬롯 이동/리사이즈
- 슬롯 hover 시 삭제 `×`
- 선택 슬롯만 `화면 맞춤` 적용
- 비율 프리셋은 선택 슬롯 기준

하단 액션:

- `화면 맞춤`
- `레이아웃 초기화`
- `취소`
- `업로드`

### 디자이너 동작 규칙

- `화면 맞춤`: 비율 무시하고 슬롯을 꽉 채움
- 비율 프리셋: 선택한 슬롯의 리사이즈 비율 고정
- 웹/날씨 소스도 일반 파일처럼 슬롯에 배치 가능
- 저장 후 다시 `수정`을 누르면 기존 배치 상태로 재편집 가능

### 날씨 콘텐츠 UX 기준

날씨 추가 팝업은 아래 필드를 가집니다.

- 위치
- 표출 형태: `가로형 / 세로형`
- 표시 유형: `단일 / 주간 / 시간별`
- 테마: `다크 / 라이트`

위치는 수기 좌표 입력이 아니라 검색 결과 선택 방식입니다.

## 스케줄 탭

### 좌측

- 스케줄 목록
- 그룹 추가
- 스케줄 추가

### 우측

- 선택 스케줄 편집
- 콘텐츠 추가
- 순서 변경
- 이름 변경 / 삭제 메뉴

### 스케줄 편집 기준

- 스케줄은 콘텐츠의 재생 순서와 길이를 다루는 곳
- 채널 연결은 여기서 하지 않음
- 그룹은 시각적 정리 목적

## 채널 탭

### 좌측

- 채널 목록
- 채널 생성

### 우측

- 선택 채널 편집
- 탭 순서: `예약 / 요일 / 기본`
- 채널별 스케줄 규칙과 색상 관리
- 달력 보기 제공

### 채널 규칙 기준

- `기본`: fallback 스케줄
- `예약`: 기간 기반
- `요일`: 주간 반복 기반
- 우선순위는 규칙 순서와 활성 상태 기준으로 해석

### 달력

- `월 / 주 / 일` 보기
- 월/주에서 날짜 클릭 시 일 보기 이동
- 일 보기 헤더는 `연 / 월 / 일` 기준
- 규칙 색상 반영

## 로그 탭

### 현재 제공 기능

- 카테고리 필터
- 검색
- 50줄 페이징
- 새로고침
- 기간 선택 다운로드

### 로그 표기

- 한 줄에 `시간 / 카테고리 / 이벤트 / 상세`
- 변경성 관리자 작업은 `actorIp` 포함

### 현재 저장 정책

- 서버 파일: `data/cms.log`
- 30일 로테이션

## 플레이어 UI

플레이어의 기준 경로는 `/client` 입니다.

### 원칙

1. 전체화면
2. 검정 배경
3. 관리자 요소 노출 금지
4. WebSocket 기반 스케줄 교체
5. 일반 파일도 필요 시 내부 디자이너 래퍼 콘텐츠로 재생 가능

### 관련 내부 페이지

- `/designer-content.html`
- `/weather-content.html`

이 두 페이지는 플레이어와 디자이너 미리보기에서 공통으로 사용됩니다.
